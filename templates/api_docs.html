Lonely Hub API Docs

Integrate mods, clients, forum, and user data into your own bots, tools, and apps.

Authentication
API tokens identify you to the API. Get yours from your Dashboard. Send the token in the X-Token header for endpoints that require it.

Some endpoints also accept the token as a token query parameter.
User API
Token required
All User API endpoints require the header X-Token: your_api_token.
POST /api/v2/upload Upload a new mod or client
HeaderTypeRequiredDescription
X-TokenstringrequiredYour API token
X-TypestringrequiredMods or Clients
X-NamestringrequiredName of the mod/client
X-DescriptionstringrequiredDescription
X-DownloadLinkstringrequiredDirect download URL
X-MCVersionstringoptionalJava, Bedrock, or All
X-VersionstringoptionalMC versions, comma-separated (e.g. 1.20.1, 1.20.4)
X-TagstringoptionalTags, comma-separated (max 10)
X-ImagestringoptionalImage URLs, comma-separated (first = banner)
X-LoadersstringoptionalLoaders, comma-separated (mods only, e.g. Fabric, Forge)
✓ Response success 
{
  "success": true,
  "id": 42,
  "url": "/mods/42/info",
  "name": "My Mod",
  "type": "mod"
}
✕ Error 
{ "success": false, "error": "X-Name is required" }

Admin API
Admin only
Admin endpoints require an API token that belongs to an admin account. Non-admin tokens will receive a 403 error.
POST /api/v3/edit Admin edit any mod or client

Same headers as /api/v2/edit but works on any mod/client regardless of ownership.

HeaderTypeRequiredDescription
X-TokenstringrequiredAdmin API token
X-TypestringrequiredMods or Clients
X-PostIdintegerrequiredID of the product to edit
X-Name, X-Description, X-Tag, X-Image, X-DownloadLinkstringoptionalFields to update
✓ Response success 
{ "success": true, "id": 42, "name": "Updated Name" }
POST /api/v3/user Manage users — ban, mute, verify, promote & more
HeaderTypeRequiredDescription
X-TokenstringrequiredAdmin API token
X-UserIdstringrequiredTarget user's ID (UUID)
X-ActionstringrequiredSee action reference below
Available Actions
X-Action valueEffect
banBlock user from logging in and accessing the site
unbanLift an existing ban
mutePrevent user from posting, commenting, or uploading
unmuteLift an existing mute
verifyGrant the verified badge (✓) to the user's profile
unverifyRemove the verified badge from the user's profile
promotePromote user to admin role
demoteRemove admin role, revert to regular user
Examples
Ban a user 
// Request headers
X-Token: your_admin_token
X-UserId: "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
X-Action: "ban"

// Response
{
  "success": true,
  "action": "ban",
  "user_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "username": "BadPlayer99"
}
Mute a user 
// Request headers
X-Token: your_admin_token
X-UserId: "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
X-Action: "mute"

// Response
{
  "success": true,
  "action": "mute",
  "user_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "username": "SpammerUser"
}
Grant verified badge 
// Request headers
X-Token: your_admin_token
X-UserId: "b9c8d7e6-f5a4-3210-bcde-fa0987654321"
X-Action: "verify"

// Response
{
  "success": true,
  "action": "verify",
  "user_id": "b9c8d7e6-f5a4-3210-bcde-fa0987654321",
  "username": "TrustedCreator",
  "badge": "verified"
}
Promote to admin 
// Request headers
X-Token: your_admin_token
X-UserId: "c3d4e5f6-a7b8-9012-cdef-ab3456789012"
X-Action: "promote"

// Response
{
  "success": true,
  "action": "promote",
  "user_id": "c3d4e5f6-a7b8-9012-cdef-ab3456789012",
  "username": "NewModerator",
  "role": "admin"
}
✕ Error (invalid action) 
{
  "success": false,
  "error": "Invalid action. Must be one of: ban, unban, mute, unmute, verify, unverify, promote, demote"
}
